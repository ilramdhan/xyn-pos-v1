# Sealed Secrets — Usage Guide

Sealed Secrets encrypts Kubernetes secrets so they can be safely committed to git.
The `sealed-secrets-controller` in the cluster is the only entity that can decrypt them.

## How It Works

```
Developer                          Cluster
─────────────────────────────      ────────────────────────────────
kubectl create secret ... \        sealed-secrets-controller
  --dry-run=client -o yaml  ─────►   │  holds master keypair
  | kubeseal > sealed.yaml           │  decrypts SealedSecret
                                     │  creates real Secret
git commit sealed.yaml ──────────────►  ArgoCD applies sealed.yaml
                                     │
                                     ▼
                               Pod reads real Secret
```

## Create a Sealed Secret

```bash
# 1. Fetch the controller's public key (do this once, commit the cert)
kubeseal --fetch-cert \
  --controller-name=sealed-secrets-controller \
  --controller-namespace=kube-system \
  > infra/k8s/base/sealed-secrets/pub-cert.pem

# 2. Create and seal a secret (example: POS service DB password)
kubectl create secret generic pos-db-secret \
  --namespace=xyn-prod \
  --from-literal=DATABASE_URL="postgres://app_user:REAL_PASSWORD@postgres:6432/xyn_pos" \
  --dry-run=client \
  -o yaml \
  | kubeseal \
    --cert infra/k8s/base/sealed-secrets/pub-cert.pem \
    --format yaml \
  > infra/k8s/overlays/prod/pos-db-sealed-secret.yaml

# 3. Commit and push — ArgoCD will apply it and the controller decrypts it
git add infra/k8s/overlays/prod/pos-db-sealed-secret.yaml
git commit -m "feat: add pos-db sealed secret for prod"
```

## Master Key Backup

The master key backup is at `infra/k8s/base/sealed-secrets/master-key-backup.yaml`
(generated by `scripts/k3s/post-install.sh`).

**Store this file in your password manager or vault — NOT in git.**

To restore the master key on a new cluster:
```bash
kubectl apply -f master-key-backup.yaml
kubectl rollout restart deployment sealed-secrets-controller -n kube-system
```

## Files in This Directory

| File | Safe to commit? |
|---|---|
| `README.md` | ✅ Yes |
| `pub-cert.pem` | ✅ Yes — public key only |
| `master-key-backup.yaml` | ❌ No — contains private key |
| `*-sealed-secret.yaml` | ✅ Yes — encrypted, that's the whole point |
